# Product

## Register

product

## Users

The primary user is Lake Blue Intelligence, a solo thinker and beginner programmer who uses ChatGPT and Gemini as daily thinking partners in Microsoft Edge. The interface is used on a real Windows desktop while conversations, files, and local tooling are active.

## Product Purpose

AGE-FX Thought Console is a local-first thinking review system. It captures visible ChatGPT and Gemini conversation turns, stores them in a D-drive SQLite database, synthesizes the previous day's thought battle record after local midnight, and recommends exactly one practical AGE equipment tool for optional Codex printing.

Success means the system disappears into the user's existing thinking habit: capture is automatic, the database remains inspectable, analysis is deep rather than transcript-like, and equipment recommendations are useful, creative, and easy to build.

## Brand Personality

Lake-blue, tactical, precise. The product should feel like a restrained AGE-FX cockpit: intelligent, calm, and technologically alive, without becoming decorative noise or a generic sci-fi dashboard.

## Anti-references

- Generic SaaS dashboards with card grids and marketing gradients.
- Decorative glassmorphism that hurts readability.
- Transparent overlays that allow desktop icons or other windows to contaminate text.
- Flat white glass panels that look like ordinary web cards with blur applied.
- Productivity guilt trackers, health trackers, todo apps, or pressure dashboards.
- UI that replays raw user transcripts instead of synthesizing thought.

## Visual Reference Constitution

The console visual language now follows a ChatGPT-web-inspired minimalist product UI: quiet white and soft gray surfaces, clear typography, restrained borders, subtle elevation, and almost no decoration. The Liquid Glass direction is retired for the production console unless the user explicitly revives it.

Non-negotiable material reading:

1. The base environment is neutral light gray, close to `#f7f7f8`, with white content surfaces.
2. Panels are ordinary product surfaces, not glass, metal, sci-fi chrome, or decorative cards. Use `#ffffff`, subtle gray borders, and very soft shadows only where hierarchy needs it.
3. Color is restrained. Keep the AGE-FX cyan brand accent, success green, and warning/error colors; avoid broad gradients, blur blobs, neon, purple glows, and glass caustics.
4. Buttons follow familiar web product controls: rounded rectangles or pills, clear hover/focus/active states, no jelly thickness, no glow.
5. Text remains practical product UI: high contrast, readable Chinese and English, consistent system sans typography.
6. The layout may keep the existing AGE-FX structure, but visual treatment should feel calm, minimal, and ChatGPT-like: sparse, readable, and trustworthy.
7. Borders are thin and functional, not decorative. Shadows are barely visible and never black-heavy.
8. The production console should never look like Liquid Glass, frosted glass, gray metal, or a sci-fi cockpit unless the user explicitly asks to revive that direction.

## Design Principles

1. Analysis over archive: the console shows synthesized thinking, not raw conversation playback.
2. Desktop-native first: the app should behave like a local Windows companion, not a web page pasted on the desktop.
3. Glass must serve legibility: transparent materials are acceptable only when text and controls remain calm and readable on a busy desktop.
4. One decisive next move: daily output culminates in exactly one equipment recommendation.
5. Local trust: data location, capture state, and API configuration should be visible and understandable.

## Accessibility & Inclusion

Target WCAG AA contrast for all functional text and controls. Respect reduced motion. Keep touch and pointer targets large enough for repeated use. Preserve Chinese and English language support. Avoid relying on color alone for state.
